  • HTML书写基本规范
    • HTML文件中只写HTML，JS、CSS尽量外引，不要写在当前页面中 ====>【样式、结构、行为三者分离，更符合W3C规范！】
    • 将link标签放在head标签的最后,并应用好CSS的层叠性。
    • 将script标签放置body结束标签之前,在不写window.onload/$(document).ready(function(){})的情况下能更好的执行JS
    • HTML标签书写要符合W3C规范的嵌套规则
    • 尽量使用HTML5语义化标签，摒弃HTML删除的标签

  • meta标签的使用
  • meta标签的作用：

    搜索引擎优化（SEO），定义页面使用语言，自动刷新并指向新的页面，实现网页转换时的动态效果，控制页面缓冲，网页定级评价，控制网页显示的窗口等！

    meta标签的组成：

    meta标签主要有两个属性，它们分别是http-equiv属性和name属性，不同的属性又有不同的参数值，这些不同的参数值就实现了不同的网页功能。

    • 一、 name属性：
      　name属性主要用于描述网页，与之对应的属性值为content，content中的内容主要是便于搜索引擎机器人查找信息和分类信息用的。
      • meta标签的name属性语法格式： 
        
    <meta name="参数"content="具体的参数值">。

    A、Keywords(关键字)　
    说明：keywords用来告诉搜索引擎你网页的关键字是什么。

    举例：
    <meta name="keywords"content="meta总结,html meta,meta属性,meta跳转">

    B、description(网站内容描述)
    说明：description用来告诉搜索引擎你的网站主要内容。

    举例：
    <meta name="description"content="薪火网研-国内领先互联网教育服务平台。">



    ===>> 以下较少使用!!!

    C、robots(机器人向导)

    说明：robots用来告诉搜索机器人哪些页面需要索引，哪些页面不需要索引。
    content的参数有all,none,index,noindex,follow,nofollow。默认是all。

    举例：
    <meta name="robots"content="none">

    具体参数如下：

    　　信息参数为all：文件将被检索，且页面上的链接可以被查询；

    　　信息参数为none：文件将不被检索，且页面上的链接不可以被查询；

    　　信息参数为index：文件将被检索；

    　　信息参数为follow：页面上的链接可以被查询；

    　　信息参数为noindex：文件将不被检索，但页面上的链接可以被查询；

    　　信息参数为nofollow：文件将被检索，但页面上的链接不可以被查询；
    • 二、 http-equiv属性：
      　　http-equiv顾名思义，相当于http的文件头作用，它可以向浏览器传回一些有用的信息，以帮助正确和精确地显示网页内容，与之对应的属性值为content，content中的内容其实就是各个参数的变量值。
      • meta标签的name属性语法格式： 
        
    <meta http-equiv="参数"content="参数变量值">；
      • http-equiv属性主要有以下几种参数： 
        
    A、Pragma(cache模式)

    说明：禁止浏览器从本地计算机的缓存中访问页面内容。

    用法：
    <meta http-equiv="Pragma"content="no-cache">

    B、Refresh(刷新)

    说明：自动刷新并指向新页面。

    用法：
    //(注意后面的引号，分别在秒数的前面和网址的后面)
    <meta http-equiv="Refresh"content="2;URL=http://www.haorooms.com">

JS/JQ规范

JS规范

  • 入口函数

  • 除一些因依赖关系需要和依赖文件放在一起的JS脚本(例如less.js=>用于编译less文件)，所有JS脚本（包括外引）应当写在 </body> 之前;
    （!!!操作DOM的JS文件，必须放在</body>之前；） ===>这样写可以避免页面加载初的页面空白

    入口函数：window.onload=function(){};所有写在入口函数中的JS代码，会等待HTML以及CSS、图片等外引资源全部加载完毕后解析并执行。 但是一个HTML中只能有一个入口函数,页面只会解析执行一个Window.onload,所以应当将将所有需要操作外部资源的JS放在入口函数中，保证JS能正常执行。 

    例：
    window.onload=function(){
          alert("页面所有资源已经加载完毕");
        }

  • 变量以及函数的命名

  • 变量命名

    变量命名应采用小驼峰命名法, 并且前缀应为名词（以此区分函数），并应当尽量在变量名中体现变量所属数据类型（例如count,length/name,title） 

    //好的命名
var maxCount = 10;
var slideTitle="mySlide";

//不好的命名(含有动词)
var setCount = 10;
var getTitle="myTitle";

    函数命名

    变量命名应采用小驼峰命名法, 并且前缀应为动词（以此区分变量） 

    常用的动词约定:
    has
    is
    get
    set
    load


//例：
// 是否可阅读
function canRead() {
    return true;
}

// 获取名称
function getName() {
    return this.name;
}

    常量

    名称全部大写，并用下划线来分割单词 

    var MAX_COUNT = 10;
var URL = 'http://www.baidu.com';

    构造函数

    大驼峰命名法=>首字母大写 

    function Student(name) {
    this.name = name;
}

var st = new Student('tom');
